# Security Policy / Политика безопасности

## Русский

FlowScope — defensive observability проект.

Сообщая об уязвимости, укажите:
1. затронутую версию/commit
2. impact
3. безопасные шаги воспроизведения
4. предлагаемое исправление (если есть)

Базовый hardening:
- смена дефолтных секретов
- ограничение внешних портов
- TLS на ingress
- регулярное обновление зависимостей

## English

FlowScope is a defensive observability project.

When reporting a vulnerability, include:
1. affected version/commit
2. impact assessment
3. safe reproduction steps
4. suggested remediation (if available)

Baseline hardening:
- rotate default secrets
- restrict exposed ports
- enable TLS at ingress
- keep dependencies up to date
